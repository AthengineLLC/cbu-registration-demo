athengine component gallery

Everything below is rendered by the real app stylesheet, served live from the product at cbu.athengine.com/proto.css. If it looks right here, it looks right in the product. Copy the markup, not the CSS.

Rule: use these class names. Do not write new CSS. If you need something that is not here, build it out of what is here, or leave a note and Ben adds a real primitive.

Buttons

Four variants, two sizes. Use <a> when it navigates, <button> when it acts.

Link styled as button 
<button class="ui-button ui-button--primary" type="button">Save</button>
<button class="ui-button ui-button--secondary ui-button--sm" type="button">Edit</button>
<a class="ui-button ui-button--primary" href="/players/new">New player</a>

variants: primary | secondary | tertiary | danger
size:     add ui-button--sm for in-row / in-table actions

Cards

The default surface. Three padding densities, optional red accent bar, optional hover lift.

Default card

Standard padding. This is what most content sits in.

Accent + compact

Red bar on the left edge. Tighter padding.

Spacious + interactive

Lifts on hover. Use when the whole card is a link.

Legacy .card

Older surface, still used by list pages and detail sections. Softer, rounder.

<section class="ui-card ui-card--pad-default">
  <header class="ui-card__header">Title</header>
  ...
</section>

padding:  ui-card--pad-compact | ui-card--pad-default | ui-card--pad-spacious
add-ons:  ui-card--accent  (left red bar)
          ui-card--interactive  (hover lift, for clickable cards)

Form controls

Wrap every control in a Field so the label, hint, and error line up.

Waivers 
<label class="ui-field">
  <span class="ui-field__label">Player name<span class="ui-field__required">*</span></span>
  <input class="ui-input" type="text">
  <span class="ui-field__hint">Helper text.</span>
</label>

error state: add ui-field--error on the label, and
             <span class="ui-field__error" role="alert">Message</span>
controls:    ui-input | ui-select | ui-textarea | ui-checkbox
grouping:    <fieldset class="ui-fieldset"><legend class="ui-fieldset__legend">
